CornStructor Help

About CornStructor

CornStructor helps inspect complex DNA and make design for Annealing-Elongation DNA construction (Gen3). It designs hierarchical construction trees, overlays fragments and oligos of the design on your reference sequence so you can see overlaps and strands at a glance, and keeps runs history.

  • Analyze: Check GC%, entropy/complexity, repeats, and stems on your reference sequence before designing to spot risky regions early.
  • Design: Set global and per-level rules, start the pipeline, and watch logs as it runs.
  • Visualize: Overlay fragments/oligos (sense/antisense), inspect overlaps and Tm, and click to see properties.
  • Recall: Reload any completed run with its note, reference sequence, and saved report in one click.
  • Persist: Reports are regenerated on demand from saved trees, GA history, and parameters.

End-to-End Design Flow

1
Prepare sequence. Paste or upload FASTA into the Sequence Editor. Invalid bases are flagged; overlays stay disabled if the sequence no longer matches the design reference.
2
Sequence analysis. Adjust analysis settings (GC, entropy, repeats, stems) to spot risky regions before designing.
3
Set design parameters. Configure construction tree globals/levels in the Parameters dialog.
4
Start design. Add an optional run description; click Start design. Logs stream immediately; artifacts land under /reports/<jobId>/.
5
Review & visualize. When complete, the sequence + tree load automatically. Choose overlay level/mode to see fragments or oligos aligned to the reference.
6
Revisit later. Open History, pick a completed run: the app restores the reference sequence, your saved note, and the report link without leaving the page.

Sequence Editor

Edit, inspect, and align design overlays against the reference sequence—built for biologists first.

  • Editing & navigation: Click to move caret; drag to select; copy/paste DNA only (non-ACGT is ignored). Keyboard: arrows/Home/End/Page-like via per-line width; Backspace/Delete mutate sequence. Invalid characters are counted and highlighted.
  • Features overlay: Turn on GC, entropy (complexity), repeats, and stems to spot risky regions. Hide the “all ranges” bar when zooming into a single range.
  • Reference integrity: Overlays only appear when the sequence matches the stored reference. Edits or a new FASTA pause overlays until you reload the matching design (or pick the run again from History).
  • Overlay modes & levels: Fragments show intermediate pieces at the chosen level; oligos show the leaf pieces. Switch levels to inspect different assembly depths.
  • Strands & styling: Sense (+) and antisense (-) bars use subtly distinct hues with uniform thickness and consistent gaps. Tooltips and labels show the fragment/oligo id.
  • Click to inspect: Click a bar to pin details: length, coordinates, strand, left/right overlaps with lengths and Tm (rounded). Selection also highlights the corresponding bar.

Sequence Analysis Parameters

These sliders govern the “quality” overlays. They are saved with each run and reapplied when you reload it.

  • GC% (window + band): Sliding window size (e.g., 100 nt) plus min/max GC thresholds per window. Smaller windows expose local spikes/dips; larger windows smooth trends. Outside-band windows are marked GC-low or GC-high.
  • Homopolymers: Minimum run length to flag stretches of identical bases; raise to reduce noise, lower to catch shorter runs.
  • Low complexity (Shannon): Entropy minimum threshold (e.g., 1.8) to flag low-diversity windows; raise to surface repetitive/biased regions, lower to tolerate repetition.
  • Repeats (tandem): K-mer range and min repetitions to catch short tandem repeats; tighten to surface dense clusters earlier.
  • Long repeats (approximate): Minimum length and percent identity for approximate repeats that can misassemble across distant loci.
  • Secondary structure (stems): Min stem length and merge gap control sensitivity to hairpins; recomputed whenever the sequence changes.

Design Tree Parameters

Construction tree settings are split into globals (apply everywhere) and levels (per-depth overrides). Set them once; they’re saved with the run.

Globals

  • Runtime controls: CPU workers fraction—governs how much parallelism is applied during design.
  • Genetic Algorithm: Population size, generations, mutation rate, crossover rate, selection pressure—control search breadth/exploitation.
  • Tm method/conditions: Computation method and environmental assumptions (e.g., salt/primer concentrations) used consistently for overlaps.

Division adjustment

  • Attempts per junction: How many refinement tries are made at each junction to improve splits.
  • Step (bp): Base-pair step size used when adjusting divisions.
  • Max total adjustment rounds: Upper bound on how many adjustment cycles the splitter performs.

Levels

  • Sizing targets: Pick the “ideal” fragment/oligo size for this level and guide how parent pieces are split.
  • Length bounds: Set the shortest and longest allowed fragments/oligos for this level.
  • Overlaps: Per-level allowed overlap ranges, Tm targets, and related penalties that shape overlap selection at that depth.

Run History

Find and restore previous designs without leaving the page.

  • Search by job id or description (note).
  • Select a completed run to reload its reference sequence, tree, saved note, and report link (overlays re-enable when the reference matches).
  • Reports open in a new tab; the current page keeps your context.
  • Delete removes the run and its linked design (optionally artifacts).

Reports & Artifacts

All artifacts live under /reports/<jobId>/ and can be re-rendered from DB snapshots.

  • index.html: entry point with links to tree, analysis, GA progress, and downloads—open this first.
  • tree.json & ga_progress.json: serialized design + GA timeline.
  • fragments.fasta & oligos.csv: sequence exports for synthesis.
  • Bundle: zipped package for offline inspection.
